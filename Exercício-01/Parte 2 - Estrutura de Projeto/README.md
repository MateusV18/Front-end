## Estrutura de Pastas e Arquivos

Abaixo está a explicação detalhada de cada componente criado nesta subpasta:

### `index.html`
É o arquivo principal da aplicação. Ele contém a estrutura robusta do documento utilizando a linguagem de marcação **HTML5**. É aqui que definimos os elementos com os seus conteúdos que aparece na página e fazemos a chamada para as folhas de estilo (CSS) e scripts (JavaScript).

### `css/`
Pasta dedicada exclusivamente à estilização visual do projeto.
* **`style.css`**: É o arquivo onde fica todo o código **CSS3**. Ele é responsável por definir as estilização da página, usando cores, fontes, layouts e design responsivo.

### `js/`
Pasta reservada para a lógica e comportamento da página.
* **`script.js`**: Arquivo que contém o código **JavaScript**. Ele é responsável por tornar a página interativa.

### `img/`
Diretório destinado ao armazenamento de todos os **arquivos de mídia estática** utilizados no projeto. Isso inclui imagens de fundo, logotipos, ícones (em formatos como PNG, JPG, SVG) ou qualquer outro recurso visual que precise ser renderizado na página.

### `README.md`
Este próprio arquivo de documentação. Escrito em formato **Markdown**, ele serve para guiar outros desenvolvedores (ou professores e avaliadores) sobre o que é o projeto, como ele está estruturado e instruções de como executá-lo.
